## 模块 1：GPU 架构与内存层次 — QA 记录
> 📅 2026-04-28 ~ 2026-04-29

**Q**：HBM 全称是什么，教程中很多缩写没有全称和翻译介绍

**A**：HBM = High Bandwidth Memory（高带宽内存）。已在模块文档开头新增术语表，包含 22 个缩写的英文全称和中文翻译，以及 6 个非缩写专有名词（Ampere/Hopper/Turing/Blackwell/DaVinci/脉动阵列）。正文中首次出现 HBM、SRAM、PCIe 时也加了内联全称标注。

---

**Q**：为什么每个 TPC 有 2 个 SM，总共 192 SM 但是实际只启用 108 个呢？

**A**：两个原因叠加：

1. **良率问题（制造原因）**：GA100 芯片面积 826mm²，7nm 工艺下 542 亿晶体管。面积越大，制造缺陷概率越高。部分 SM 可能有缺陷，硬件做法是禁用坏的 SM，启用完好的部分（叫 "harvesting"）。

2. **产品线划分（商业原因）**：NVIDIA 基于同一颗 GA100 芯片，通过启用不同数量的 SM 划分产品等级：
   - A100 SXM/PCIe: 108 SM（旗舰）
   - A30: 56 SM（中端）
   - A40: 84 SM（专业可视化）
   
   同一颗芯片，禁用更多 SM 就变成低端产品，提高整体利润。实际到手的 A100 中，有些 SM 可能没有缺陷，只是被人为禁用来区分产品线。

**Q**：什么是脉动阵列架构，如何简单理解这种架构的工作，并且为什么这种架构能单周期内处理不同的精度数据？

**A**：

**脉动阵列（Systolic Array）原理**："Systolic" = 心脏搏动的。数据像血液一样，在处理单元（PE）之间有节奏地"脉动"流动。

矩阵乘法执行过程：
- A 的元素沿水平方向流动，B 的元素沿垂直方向流动
- 每个 PE 在每个时钟周期做 1 次乘法 + 1 次加法（MAC）
- 32×32 阵列 = 每周期 1024 次 MAC

直觉类比：A 的行数据像水从左边流进来，B 的列数据像水从上面流下来，每个交叉点（PE）把两股水"混合"（乘加），底部流出的就是 C 矩阵结果。

**为什么能处理不同精度**：不是同一个电路同时处理不同精度，而是同一块芯片面积上集成了多种精度的专用电路：
- FP16 乘法器 ≈ FP32 的 1/4 面积 → 同面积放 4 倍数量 → 吞吐量 4×
- INT8 乘法器 ≈ FP32 的 1/16 面积 → 同面积放 16 倍数量 → 吞吐量 16×

核心原理：精度越低 → 电路越小 → 同面积放更多计算单元 → 并行度越高 → 吞吐量越大。程序运行时通过指令选择精度，硬件路由到对应电路。

**Q**：全局内存带宽 2,039 GB/s ≈ 2 TB/s，和 L2 缓存的 ~3 TB/s 差距不大，为什么说 L2 比全局内存快很多？

**A**：带宽差距确实只有 1.5 倍，但真正拉开差距的是**延迟**和**共享方式**：

1. **延迟差距**：L2 200 周期 vs HBM 500 周期 → 2.5 倍差距。随机访问模式下延迟影响远大于带宽。

2. **带宽是"共享"还是"独占"**：
   - 全局内存 2 TB/s → 108 个 SM 共享 → 每 SM 约 18.5 GB/s
   - L2 缓存 3 TB/s → 108 个 SM 共享 → 每 SM 约 28 GB/s
   - 共享内存 1 TB/s/SM → SM 独占 → 每 SM 1 TB/s

   每 SM 实际可用带宽：共享内存 vs 全局内存 → 实际 ~54 倍差距。

3. **L2 的真正价值**是缓存命中：权重被 108 个 SM 反复读取时，无 L2 则 108 次 HBM 访问，有 L2 则只需 1 次。

### Web Search 补充内容

**1. TPU MXU 脉动阵列规模演进**：
- TPUv2~v5: MXU 128×128，每周期 16,384 次乘加
- TPU v6e 起: MXU 扩展到 256×256，每周期 65,536 次乘加
- v5p 和 TPU7x 引入 SparseCore（每芯片 4 个），加速稀疏操作
- MXU 脉动阵列在矩阵乘法期间**无需访存**——数据在 MAC 单元网格中以波浪式流动
- 来源：Google Cloud TPU 官方架构文档

**2. 屋顶线模型（Roofline Model）**：
- 由 Samuel Williams、Andrew Waterman、David Patterson 于 2008 年提出
- 脊点（Ridge Point）= 计算屋顶与内存屋顶交点的 X 坐标，代表脱离内存瓶颈所需的最小操作强度
- 随着计算增长远超内存带宽增长，脊点不断右移
- 内存墙概念由 Wulf & McKee 于 1994 年提出
- LLM 推理的 Decode 阶段算术强度极低（约 1 FLOP/byte），严重 memory-bound
- 来源：Modal GPU Glossary

**3. Warp 调度细节**：
- SM 顺序发射指令，**不做分支预测**，**不做推测执行**（与 CPU 不同）
- Warp 执行上下文（PC、寄存器）在整个生命周期内**常驻片上** → Warp 切换**零开销**
- A100 支持 Compute Capability 7.0+ 的"独立线程调度"（Independent Thread Scheduling）
- 每个线程有独立的 PC 和调用栈，线程可以在 sub-warp 粒度上发散和重汇合
- 但需要显式 `__syncwarp()` 同步
- 来源：NVIDIA CUDA Programming Guide
